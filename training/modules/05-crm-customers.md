# Module 05 — CRM & Customers / إدارة العملاء

> **Duration:** 20 minutes | **Audience:** Cashier, manager
> **Route:** `/crm`

---

## What this module covers / ماذا يغطي

The **CRM** (Customer Relationship Management) module stores your pharmacy's customer records. Every invoice needs a client — so keeping customers organized here makes sales faster.

وحدة **إدارة العملاء** تخزّن سجلات عملاء الصيدلية. كل فاتورة تحتاج عميل — لذلك تنظيم العملاء هنا يسرّع المبيعات.

---

## 1. Client list / قائمة العملاء

Open **CRM / العملاء** from the sidebar.

| Column | Arabic | Meaning |
|---|---|---|
| Name | الاسم | Customer name |
| Phone | الهاتف | Mobile number |
| Email | البريد | Email (optional) |
| Status | الحالة | Active, lead, inactive |
| Company | الشركة | For corporate accounts |

---

## 2. Adding a customer / إضافة عميل

1. Click **Add Client / إضافة عميل**
2. Fill in:

| Field | Arabic | Example |
|---|---|---|
| Name | الاسم | أحمد محمد علي |
| Phone | الهاتف | 01012345678 |
| Email | البريد | (optional) |
| Company | الشركة | (for corporate accounts) |
| Notes | ملاحظات | "عميل دائم — حساسية بنسلين" |
| Status | الحالة | Active |

3. Click **Save**

> **Tip for pharmacies:** Add allergy notes in the Notes field — useful for pharmacists reviewing customer history.

---

## 3. Searching customers / البحث عن عميل

- Use the **search box** to find by name or phone
- Type partial phone: "0101234" finds 01012345678
- Arabic and English names both work

---

## 4. WhatsApp contact / التواصل عبر واتساب

From any client card:
1. Click the **WhatsApp** button (green icon)
2. Opens WhatsApp with the customer's phone number pre-filled
3. Send reminders, invoice copies, or health tips

> Phone number must be in Egyptian format (01xxxxxxxxx) for WhatsApp links to work.

---

## 5. Linking customers to invoices / ربط العملاء بالفواتير

When creating an invoice (Module 03):
1. The **Client** dropdown pulls from this CRM list
2. Select the customer → their name and phone auto-fill on the invoice
3. Invoice history builds up per customer over time

---

## Walk-in customers / عملاء بدون حجز

For customers who don't have a record yet:

**Option A (recommended):** Quick-add in CRM first, then create invoice
**Option B:** Create invoice and select "Walk-in" if you have a generic walk-in client set up

> Best practice: Create a generic "عميل كاونتر" (counter customer) record for anonymous walk-ins, and named records for regulars.

## Common mistakes / أخطاء شائعة

| Mistake | Fix |
|---|---|
| Duplicate customers | Search before adding — same person with 2 records |
| Wrong phone format | Use 01xxxxxxxxx (11 digits, no +20 prefix) |
| No client on invoice | Invoice creation requires a client — add one first |
| Ignoring notes field | Use it for allergies, preferences, insurance info |
